# Implementation Plan: Offline-First with Auto-Sync

This document outlines the strategy for making **HisabKhata** work offline using IndexedDB for local persistence and a background synchronization layer for Firebase.

## 1. Technology Stack
- **PWA Support**: `vite-plugin-pwa` for service worker management and asset caching.
- **Local Database**: `idb` (lightweight IndexedDB wrapper) or `Dexie.js` for structured local storage.
- **Sync Logic**: Custom `SyncManager` using the `online`/`offline` browser events.

## 2. Architecture: The "Sync Layer"

Current Flow:
`UI -> Firebase (Direct)`

Proposed Flow:
`UI -> Local Database (IndexedDB) -> Sync Manager -> Firebase`

### Key Components:
1. **Local Store**: Mirrors the Firebase structure (`customers`, `transactions`).
2. **The Outbox**: A dedicated table in IndexedDB that stores operations (`CREATE`, `UPDATE`, `DELETE`) that haven't reached the server yet.
3. **Connectivity Observer**: Monitors `window.ononline` and `window.onoffline`.

## 3. Detailed Steps

### Step 1: PWA Integration
Install and configure `vite-plugin-pwa`.
```bash
npm install vite-plugin-pwa -D
```
Configure `vite.config.js` to enable `registerType: 'autoUpdate'` and define icons/manifest.

### Step 2: Database Service Refactor (`firebase.js`)
Instead of calling Firebase methods directly, the `dbService` will:
1. Generate a unique ID (UUID) locally.
2. Save the data to IndexedDB.
3. Add a record to the `outbox` table.
4. Trigger the `SyncManager`.

### Step 3: The Sync Manager
A background process that:
1. Checks if the navigator is online.
2. Reads the oldest entry from the `outbox`.
3. Executes the corresponding Firebase operation.
4. On success, removes the entry from the `outbox`.
5. On failure (network error), waits for the next `online` event.

### Step 4: Data Reconciliation (Pulling from Cloud)
When the app starts or comes online:
1. Fetch latest data from Firebase.
2. Merge with local IndexedDB data.
3. **Conflict Resolution**: "Last Write Wins" based on a `updatedAt` timestamp.

## 4. UI/UX Requirements
- **Status Indicator**: A small cloud icon in the navbar showing:
    - `Synced` (Green check)
    - `Offline` (Gray cloud)
    - `Syncing` (Rotating arrows)
- **Optimistic UI**: The app must not show loading spinners during data entry. Changes should appear in the list immediately.

## 5. Potential Challenges
- **Firebase Security Rules**: Ensure the `userId` is validated correctly during background sync.
- **Image/Asset Storage**: If HisabKhata allows attachments, these need to be stored as Blobs in IndexedDB while offline.
